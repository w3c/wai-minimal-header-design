WCAG 2 Guidance

Summary

This page shows the different kinds of guidance that exist for the Web Content Accessibility Guidelines (WCAG) 2. It helps you know where to go for which type of information.

For background and an introduction to WCAG, see Web Content Accessibility Guidelines (WCAG) Overview.

Types of guidance

Sufficient Techniques

Sufficient Techniques are examples of ways to meet a WCAG success criterion. If you follow a Sufficient Technique for a given WCAG Success Criterion, you will meet that criterion.

  • Sufficient Technique H48
    Using ol, ul and dl for lists or groups of links
    Sufficient to meet 1.3.1: Info and relationships
  • Sufficient Technique G115
    Using semantic elements to mark up structure
    Sufficient (if combined) to meet 1.3.1: Info and relationships

Advisory Techniques

Advisory Techniques are examples of accessibility best practices.

  • Advisory Technique G115, due to
    Using semantic elements to mark up structure
    A best practice for 1.3.1: Info and relationships

Failure Techniques

Failure Techniques are examples of what would constitute a failure to meet a WCAG success criterion.

  • Failure F2, due to
    Using changes in text presentation to convey information without using the appropriate markup or text
    Failure to meet 1.3.1: Info and relationships

Understanding Documents

Understanding Documents explain a WCAG Succes Criterion.

  • Understanding
    2.4.3 Focus Order
  • Understanding
    2.4.3 Focus Order (with quote)
  • Understanding
    2.4.3 Focus Order (with quote and video)

COGA Design Patterns

COGA Design Patterns are examples of how to improve your website for people with cognitive disabilities.

  • Design Pattern
    Make the Purpose of Your Page Clear

ACT Rules

ACT Rules are WCAG Success Criteria interpreted as test rules.

  • ACT Rule
    HTML page has title
    To test 2.4.2: Page Titled

Getting from one document to the other

The WCAG 2 documents are interlinked. For example:

  • In WCAG 2,
    • Links such as “How to Meet 1.1.1” go to the How to Meet WCAG 2 document, and jumps down to that section within the document.
    • Links such as “Understanding 1.1.1” links go to an Understanding WCAG 2 topic page.
      • You can get to the Contents list of all the Understanding topics from the [Contents] button at the top.
      • You can get all the topics in one large web page from the “single HTML version” link in the footer of each topic page.
  • In both How to Meet WCAG 2 and Understanding WCAG 2, there are lists of techniques that are links. These links go to a Techniques for WCAG 2 topic page.
    • You can get to the Contents list of all the Techniques topics from the [Contents] button at the top.
    • You can get all the topics in one large web page from the “single HTML version” link in the footer of each topic page.
